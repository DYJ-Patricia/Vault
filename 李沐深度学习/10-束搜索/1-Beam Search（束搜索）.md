# 贪心搜索

![[Pasted image 20260315143047.png]]

- 在 seq2seq 中我们使用了贪心搜索来预测序列
    
    - 将当前时刻预测概率最大的词输出
    
- 但贪心很可能不是最优的：

贪心：`0.5×0.4×0.4×0.6=0.048`

很好的选项：`0.5×0.3×0.6×...=0.054`

# 穷举搜索

![[Pasted image 20260315143335.png]]

- 最优算法：对所有可能的序列，计算它的概率，然后选取最好的那个
- 如果输出字典大小为 n，序列最长为 T，那么我们需要考察 nT 个序列
    - n=10000, T=10:  nT=1040
    - 计算上不可行



# 所以要在两者之间另找一种方法（束搜索）

![[Pasted image 20260315144333.png]]

- 保存最好的 k 个候选
- 在每个时刻，对每个候选新加一项（n 种可能），在 kn 个选项中选出最好的 k 个


# 复杂度&分数

- log是给长句子好处，因为句子越长概率越小，不要一直找短句子

![[Pasted image 20260315145106.png]]


# **总结**

- 束搜索在每次搜索时保存 k 个最好的候选
    - k=1 时是贪心搜索
    - k=n 
    - 穷举是个更复杂的case
